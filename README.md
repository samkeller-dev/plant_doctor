# plant_doctor

A **local-first RAG diagnostic API** for houseplant problems. Describe a symptom; receive a structured JSON diagnosis with citations, confidence, recommended actions, and caveats — all generated by a self-hosted **Mistral 7B Instruct** model with **prompt-injection guardrails**, **off-topic rejection**, and **citation validation** baked in.

This repo is a focused demonstration of a small, well-guarded retrieval-augmented-generation stack: nothing leaves the host, every claim is grounded in a curated corpus, and the model cannot quietly cite sources that do not exist.

---

## Tech stack

- **Ollama** — self-hosted local model runtime
- **Mistral 7B Instruct** (`mistral:7b-instruct`) — the LLM
- **nomic-embed-text** — the embedding model (also via Ollama)
- **LlamaIndex** — ingestion + retrieval orchestration
- **ChromaDB** — persistent vector store
- **FastAPI** — async HTTP API
- **Pydantic v2** — strict JSON-schema validation on the model's output
- **Docker** + **docker-compose** — one-command stack
- **pytest** — unit + adversarial test suite

```
                 └──────────────────────────────────────┘
```

## Quickstart

### With Docker (recommended)

```bash
cp .env.example .env
docker compose up -d --build
# wait for ollama-init to finish pulling models (~4.7 GB on first run)
docker compose run --rm app python -m scripts.ingest
docker compose restart app   # app cold-starts before Chroma exists; restart picks up the index
```

The app is now live at `http://localhost:8000`.

### Without Docker

```bash
# Install Ollama from https://ollama.com and start it
ollama pull mistral:7b-instruct
ollama pull nomic-embed-text

python -m venv .venv && source .venv/bin/activate   # or .\.venv\Scripts\activate on Windows
pip install -r requirements.txt
cp .env.example .env
# Edit OLLAMA_BASE_URL=http://localhost:11434 in .env
python -m scripts.ingest
uvicorn app.main:app --reload
```

## Example: a normal query

```bash
curl -s -X POST http://localhost:8000/ask \
     -H 'Content-Type: application/json' \
     -d '{"description": "monstera leaves yellowing, weekly watering, low light", "plant_type": "Monstera deliciosa"}'
```

Response (illustrative — exact wording varies per model run):

```json
{
  "diagnosis": "Likely overwatering combined with insufficient light, progressing toward root rot.",
  "confidence": "medium",
  "evidence_cited": [
    {
      "source": "root_rot_overwatering.md",
      "quote": "Lower leaves yellow and drop quickly. Wilting despite moist soil.",
      "relevance": "primary"
    },
    {
      "source": "light_requirements.md",
      "quote": "Soil stays wet for much longer than expected — a low-light plant uses less water.",
      "relevance": "supporting"
    }
  ],
  "recommended_actions": [
    {
      "action": "Stop watering and let the top 2 inches of soil dry; gently unpot to inspect roots.",
      "urgency": "immediate",
      "rationale": "Saturated soil is the proximate trigger for root rot."
    },
    {
      "action": "Move the plant closer to a bright indirect light source.",
      "urgency": "this_week",
      "rationale": "Low-light plants use less water; the watering schedule must match the light."
    }
  ],
  "differential_diagnoses": [
    "Natural senescence of the oldest leaves",
    "Nutrient deficiency masked by waterlogged roots"
  ],
  "caveats": [
    "Without inspecting the roots directly, distinguishing early overwatering from established root rot is uncertain."
  ]
}
```

```json
  }

```

## Guardrails

Three layers, each independently testable.

| Layer | Attack class | Defence | On hit |
|---|---|---|---|
| 1 | Prompt injection (`ignore previous instructions`, role-switch, system-tag spoofing, jailbreak prompts) | Regex blocklist over ~12 well-known phrasings | `400 prompt_injection` with the matched pattern |
| 2 | Off-topic abuse (cooking, code, medical, politics, etc.) | Cosine similarity of the query embedding against the corpus centroid; reject below `OFFTOPIC_SIM_THRESHOLD` | `400 off_topic` with similarity score |
| 3 | Citation hallucination (model invents a source filename) | Output is parsed into the `Diagnosis` Pydantic schema, then every `evidence_cited[*].source` is checked against the actual set of corpus filenames. One re-prompt on violation; second strike returns `502` with the offending response. | `502 invalid_citation` (or `502 model_invalid_json` if the JSON itself was malformed) |

The output JSON schema is enforced both by Ollama's `format=json` mode AND by Pydantic v2 validation server-side — the model cannot return a shape the client did not opt in to.

## Tests

```bash
pip install -r requirements.txt
pytest -q
```

Tests are designed to run **without** a live Ollama or Chroma instance: the off-topic detector takes an injected embedder, the citation validator takes a constructed `Diagnosis`, and the schema tests exercise Pydantic round-tripping.

## Limitations

- The corpus is small and intentionally curated; this is not a substitute for a horticulturist or extension diagnostician on a serious plant problem.
- Off-topic detection is similarity-based; it is robust to obvious cases but a determined attacker who frames their request in plant vocabulary may slip through. Layer 3 still constrains the output to real corpus citations regardless.
- Mistral 7B Instruct in 4-bit quantization runs comfortably on a modern laptop CPU but is slow; expect ~10–30s per request. Swap in a hosted frontier model for lower latency at the cost of locality.
- No streaming — the API returns the full validated JSON or an error.

---

## Why this exists

This repo is a portfolio demonstration: a small, complete, well-guarded local-RAG system showing the wiring between **Ollama**, **LlamaIndex**, **ChromaDB**, **FastAPI**, **Pydantic**, and **Docker**, with **pytest** coverage of the guardrail surface. The houseplant domain is incidental; the engineering pattern generalizes to any narrow-domain assistant where ungrounded answers and prompt injection are unacceptable.
